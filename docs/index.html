Documentation: XMCLib for Roblox UI Development

Last updated: 2026-03-17 • Educational documentation

1. Installation

  1. Create a ModuleScript named XMCLib in ReplicatedStorage.
  2. Paste the library source code.
  3. Require it from a LocalScript and initialize your UI window.

2. Basic window creation

Use a single root window to centralize tabs and controls. Keep a clear naming convention for each section so your team can navigate the UI code quickly.

3. Core components

  • Buttons: Trigger actions with clear callback naming.
  • Toggles: Store boolean state in one shared state table.
  • Sliders: Validate min/max ranges and default values.
  • Dropdowns: Prefer readable labels and stable option IDs.
  • Notifications: Use them for feedback, not for persistent logs.

4. Recommended project structure

Split UI logic into modules: UIState, Actions, and Theme. Keep XMCLib bindings thin so game logic stays independent of presentation code.

5. Accessibility and usability notes

Design with sufficient contrast, clear text labels, and predictable spacing. For mobile players, avoid tiny touch targets and test every interaction with touch input.

6. Learning path

For deep practical training, continue with the complete XMCLib full tutorial and the best practices article.
